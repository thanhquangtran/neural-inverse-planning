\section{Project Status So Far}
We originally aimed at richer Bayesian inverse planning settings, but the
instructor feedback and implementation risk pushed us toward a simpler
gridworld where exact inference is still tractable. This gives us a reliable
reference model and lets us spend the remaining time on analysis and model
comparison instead of full belief-space POMDP reimplementation.

So far, we have implemented the main pipeline: a deterministic gridworld task,
an approximately rational agent, exact online goal inference, synthetic dataset
generation, and three learned approximations: \textit{Action-GRU},
\textit{Conv-GRU}, and \textit{Goal-Conditioned Policy}. We also assembled a
working \texttt{demo.ipynb} notebook with qualitative examples, scaling
experiments, rationality ablations, and held-out comparisons.

\section{Task Setting}
Our current environment is a deterministic $8 \times 8$ gridworld with walls,
one fixed start state, and three candidate goals. The default start location is
the lower-left cell $(7,0)$, and the three goals are placed at $(0,0)$,
$(0,7)$, and $(7,7)$. The agent chooses from five actions: stay, up, down,
left, and right. Invalid moves do not change the state. For the current exact
observer and the main notebook experiments, we use a Boltzmann-rational agent
with $\beta = 2.0$ and typically evaluate trajectories of length 12.

This setting is simple but still produces real ambiguity early in a trajectory.
At the same time, it is small enough that exact inverse planning over the three
candidate goals remains tractable, making it a useful reference task.

\section{How Is The Data Obtained?}
All data are generated synthetically from the exact task model. For each
episode, we sample one latent goal uniformly, roll out a trajectory from the
fixed start state under the Boltzmann policy for that goal, and then run the
exact observer on the resulting action sequence.

This gives both the final posterior and the online posterior after each action,
which are used as supervision targets. Each timestep is encoded as a
six-channel tensor: walls, agent position, three goal channels, and normalized
time. The dataset stores positions, action indices, sampled goal, final exact
posterior, online posterior trace, and goal-conditioned inputs for the policy
model.

\subsection{Action-GRU}
At each timestep, the model receives the grid tensor and the previous action.
The grid is flattened into a feature vector, the previous action is embedded,
and the concatenated vector is passed into a GRU. The last hidden state is used
to predict the final goal, while all hidden states produce posterior logits
over goals at every step. This is the simplest sequence baseline because it
asks whether a generic recurrent model can infer goals from observation-action
history without an explicit spatial encoder.

\subsection{Conv-GRU}
This model keeps the same recurrent structure and outputs as
\textit{Action-GRU},
but changes how each frame is processed. Instead of flattening the grid
immediately, a convolutional encoder extracts spatial features before they are
concatenated with the previous-action embedding and passed into the GRU. The
output is again a final-goal prediction plus posterior logits by timestep. The
design goal is to preserve local spatial relationships such as walls, free
space, and goal positions.

\subsection{Goal-Conditioned Policy}
This model takes the grid tensor, a candidate goal identity, and the previous
action as input at each timestep. The grid is encoded spatially, the goal and
previous action are embedded, and the combined features are passed into a GRU
that outputs action logits rather than goal logits. At evaluation time, we run
the model once per candidate goal, score the observed trajectory under each
goal-conditioned action model, and normalize those scores into a posterior over
goals. This design is closer to inverse planning because it learns an action
model that is inverted afterward.

\section{Results}
\subsection{Which learned model works best on the main held-out evaluation set?}
The current results are encouraging. On the held-out evaluation set, trained
with the largest current budget (4096 training episodes), \textit{Action-GRU}
reaches final-goal accuracy 0.9678 with mean posterior KL 0.0916,
\textit{Conv-GRU} reaches 0.9609 accuracy with KL 0.0953, and
\textit{Goal-Conditioned Policy} reaches 0.9824 accuracy with KL 0.0318.
Posterior KL is the more relevant metric here because it measures how well a
model tracks the exact observer over time, not just whether it predicts the
correct final goal. By that metric, \textit{Goal-Conditioned Policy} is the
strongest learned approximation so far.

\subsection{How much training data does each model need?}
The data-budget study in the notebook shows the same pattern. At 64 training
episodes, all methods are still far from the exact observer, with mean
posterior KL between 0.5908 and 0.6510. By 1024 episodes, the
 \textit{Goal-Conditioned Policy} improves to 0.0478 mean KL, while
\textit{Action-GRU} and \textit{Conv-GRU} remain at 0.1460 and 0.1685. This
suggests that modeling action generation conditioned on the candidate goal is a
more data-efficient way to approximate inverse planning than directly mapping a
trajectory to a goal label.

\subsection{How does agent rationality affect inverse-planning difficulty?}
The rationality ablation is also informative. When the acting agent is more
rational ($\beta = 3.0$), \textit{Goal-Conditioned Policy} achieves mean
posterior KL 0.0226, much better than the direct classifiers. At lower
rationality ($\beta = 0.75$), all models are closer together, with KL values
between 0.0627 and 0.0844. This is consistent with the interpretation that
noisier action sequences contain weaker evidence about the latent goal, so the
inference problem itself becomes less separable.

\subsection{Do the learned models track the exact posterior on a held-out example?}
The held-out trajectory comparison is also useful because it shows that all
three learned models can often identify the correct final goal even when their
intermediate posterior paths differ. On one held-out sequence in the notebook,
the exact final posterior is approximately $[0.982, 0.018, 0.000]$, while the
three learned models all place almost all probability mass on the correct goal
by the end. This is encouraging for final prediction, but it also reinforces
why we need the prefix-level KL plots: the interesting question is not just
whether the model is correct at the end, but whether it reasons in a way that
tracks the exact observer throughout the trajectory.

\section{Discussion}
The results also surfaced useful negative findings. The conv-GRU has not yet
outperformed the simpler \textit{Action-GRU}, so adding spatial structure alone
is not enough in this small environment. In addition, the
rationality ablation shows that performance depends noticeably on the agent's
Boltzmann rationality parameter $\beta$, which means we should be careful about
claiming that one learned model is uniformly best across all behavior regimes.
Finally, we have not yet implemented Bayesian inverse planning in belief space
for a true POMDP, and based on the instructor feedback this remains too risky
to treat as the core deliverable for this semester timeline.
